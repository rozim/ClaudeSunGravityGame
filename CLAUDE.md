# Git Workflow
- Always create a new branch before making any changes
- Branch naming: feature/short-description or fix/short-description
- Never commit or merge a branch directly to main without being asked.
- Make a change and then ask the user to run the app and evaluate it, and keep git in the branch.

# Workflow
Read GOAL.md.

If PLAN.md doesn't exist, analyze the goal from GOAL.md and create a detailed
plan with [ ] checkboxes.
Show me the plan and wait for approval before executing.
After every step is complete add a line to LOG.md stating what step you completed and how long it took.
Only append to LOG.md, do not overwrite it.

If GOAL.md and PLAN.md exist, show current status and resume from first incomplete step.
